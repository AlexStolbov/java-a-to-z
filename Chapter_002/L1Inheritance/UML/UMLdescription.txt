Описание классов.

Request - Общий класс заявка. Содержит общие поля и методы для всех заявок. От него наследуются классы для различных видов заявок.
	Поля:
	name - имя заявки
	description - описание заявки
	comments -  массив комментариев (или другой тип для множества) содержит объекты типа Comment. Т.е. одна заявка может содержать много комментариев
	Методы:
	editRequest(descr) - редактирование заявки. Пока только устанавливает описание заявки. Будет перекрываться в наследниках.
	addComment() - добавляет новый комментарий в заявку

Наследники Request. Реализуют особенности различных видов заявок. Предполагаю, что разные виды заявок нужно будет обрабатыавть по разному.
	Question - заявка вопрос.
	Bug - заявка об ошибке.

Comment - класс комментарий к заявке. Комментарий со временем может обрасти доп. полями и методами, поэтому в отдельный класс.

Tracker - класс, содержащий все заявки и обеспечивающий работу с ними, как со множеством. 
	Поля:
	requests - массив всех заявок, объекты типа Request
	Методы:
	addRequest() - Добавляет новую заявку
	deleteRequest() - Удаляет заявку, 
	getListRequest() - Формирует список заявок с фильтром или без.

Display - визуализация и взаимодействие с пользователем. Отображает необходимую информацию, получает ввод от пользователя.
	Поля:
	tracker - объект, который визуализируем
	Методы:
	displayListRequest() - показать список заявок
	getActiveFromUsers() - показать меню действий, получить выбор пользователя.
